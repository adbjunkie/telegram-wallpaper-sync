# Server + Bot (Python)

This directory contains everything needed to receive photos from Telegram and expose them to the Android app.

It is designed to deploy cleanly to **Railway** from a GitHub repo (webhook mode + optional persistent Volume).

## Quick run (local)

```bash
cd server

# 1. Create venv (once)
python -m venv venv
source venv/bin/activate          # macOS/Linux
# venv\Scripts\activate           # Windows

# 2. Install
pip install -r requirements.txt

# 3. Configure
cp .env.example .env
# Edit .env and set TELEGRAM_BOT_TOKEN + PUBLIC_BASE_URL (use ngrok https for local phone testing)

# 4. Run
python main.py
# or
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

The process starts both:
- The Telegram bot (long polling by default, or webhook if a public https PUBLIC_BASE_URL is set)
- FastAPI on the configured port

## Expose to your phone (local dev)

Use ngrok (free):

```bash
ngrok http 8000
```

Take the `https://...ngrok.io` URL and:
- Put it in `.env` as `PUBLIC_BASE_URL`
- Put the same URL in the Android app as `BASE_URL`

## Deploying to Railway (GitHub)

See the main project [README.md](../README.md) section **"Deploying to Railway + GitHub"**.

Key points for Railway:
- Use the provided `Dockerfile` or `Procfile`
- Set `TELEGRAM_BOT_TOKEN` and `PUBLIC_BASE_URL` (https://xxx.up.railway.app) in Variables
- **Strongly recommended**: Add a Volume mounted at `/data` and set `DATA_DIR=/data`
- The code will automatically prefer webhook mode on https public URLs (no constant polling)

## What the bot does

- `/start <device_id>` — links your Telegram chat to that Android device (the link is generated by the app)
- Any photo you send (or forward) to the bot → downloaded and registered as "pending" for the linked device
- When the Android app calls POST /apply → the bot automatically sends the photo back to you in Telegram with a success message

## Files

- `main.py` — FastAPI routes + all bot handlers + lifespan that runs polling
- `database.py` — SQLite helpers (links + pending + history)
- `requirements.txt`
- `.env.example`

Images land in `received_images/` (served at `/images/...`).

## Production notes
